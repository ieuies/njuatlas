# NJUAtlas 部署指南

本文档说明 NJUAtlas（njuatlas.cn）的本地运行与云端部署流程。本地部署适用于开发调试与功能验证；生产环境基于 Render + Neon PostgreSQL + 阿里云 DNS。

---

## 一、环境要求

| 依赖 | 说明 |
|------|------|
| **Python 3.10+** | 后端运行时；`requirements.txt` 锁定 **Flask 3.1.3**（2026-02 稳定版，官方安装：`pip install Flask`） |
| **PostgreSQL（可选，本地可用 SQLite）** | 关系型数据库；生产环境使用 **Neon** 托管实例，连接须启用 TLS |
| **网络连接** | 用于 pip 依赖安装及 Neon、高德、大模型 API 等外部服务访问 |
| **Windows PowerShell** | 执行 `start.ps1` 本地一键启动脚本（仅 Windows） |

环境变量通过 `backend/.env` 文件或操作系统环境变量注入，由 `python-dotenv` 在应用启动时加载；敏感配置不写入代码仓库。

## 二、本地部署

### 2.1 获取源码

```bash
git clone <仓库地址> njuatlas
cd njuatlas
```

### 2.2 安装 Python 依赖

```bash
cd backend
python -m venv venv

# Windows PowerShell
.\venv\Scripts\Activate.ps1

pip install -r requirements.txt
```

### 2.3 配置环境变量

在 `backend/` 目录复制模板并编辑：

```powershell
Copy-Item .env.example .env
```

至少需要配置以下变量（向项目组成员索取，或自行申请相应 API Key）：

| 变量名 | 用途 |
|--------|------|
| `SECRET_KEY` | Flask 会话与 JWT HS256 签名密钥（生成：`python -c "import secrets; print(secrets.token_urlsafe(32))"`） |
| `GAODE_API_KEY` | 高德地图 Web API Key |
| `BAILIAN_API_KEY` 或 `ZHIPU_API_KEY` | 大模型 API（可二选一或同时配置，`LLM_PROVIDER` 控制调用优先级） |
| `DATABASE_URL` | （可选）Neon PostgreSQL 连接串；未配置时本地默认使用 SQLite：`backend/foodmap.db` |

#### Neon 连接规范（2026 官方要求）

- 所有连接须启用 SSL/TLS，连接串包含 `?sslmode=require`
- **应用运行时**推荐使用 PgBouncer **池化连接**（主机名含 `-pooler` 后缀），适用于 Web 服务并发场景
- **Schema 迁移**（`flask db upgrade`）推荐使用 **直连**（主机名不含 `-pooler`），规避 PgBouncer 事务模式对 `SET`、临时表等操作的限制

示例（请替换为 Neon 控制台提供的实际连接串）：

```env
# 应用运行时（池化连接）
DATABASE_URL=postgresql://user:pass@ep-xxxx-pooler.region.aws.neon.tech/dbname?sslmode=require

# 迁移失败时可临时切换为直连 URL 执行 flask db upgrade
```

`backend/app/config.py` 会将 `postgres://` / `postgresql://` 规范化为 `postgresql+psycopg://`（psycopg 3.x 驱动）。

### 2.4 启动服务

#### 方式 A：使用 `start.ps1`（Windows）

```powershell
cd D:\njuatlas

# 若遇脚本执行策略限制，仅对当前进程临时放行（不修改 LocalMachine 策略）
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass

.\start.ps1
```

`start.ps1` 行为说明：

- 进程 1：`backend/` 目录执行 `python run.py`，后端监听 **http://localhost:5000**
- 进程 2：项目根目录执行 `python -m http.server 8080`，前端静态资源 **http://localhost:8080**
- 脚本不自动创建虚拟环境，也不加载 `.env`；须预先完成依赖安装与环境变量配置，并确保 `python` 指向已安装依赖的解释器。

Flask 官方支持 `flask run` 启动开发服务器；本项目本地入口为 `python run.py`（`app.run(debug=True)`），以项目脚本为准。

#### 方式 B：手动分步启动（跨平台）

```bash
# 终端 1：后端
cd backend
python run.py

# 终端 2：前端静态服务
cd ..
python -m http.server 8080
```

### 2.5 本地 API 联调

默认情况下，`http://localhost:8080` 前端请求指向生产 API。若需对接本地 Flask 实例，在 `index.html` 的 `NJUATLAS_CONFIG` 中设置：

```javascript
window.NJUATLAS_CONFIG.USE_LOCAL_API = true;
```

### 2.6 访问验证

浏览器访问 **http://localhost:8080**。注册流程依赖邮件服务；未配置 `RESEND_API_KEY` 时，验证码输出至后端终端日志，仅适用于本地调试。

## 三、云端部署（Render + 阿里云）

### 3.1 Render Blueprint 配置

> Blueprint 中 `env: python` 字段已废弃，须使用 **`runtime: python`**。本项目 `render.yaml` 已采用当前规范。

`render.yaml` 定义三类资源：

| 服务 | 类型 | 说明 |
|------|------|------|
| `njuatlas-frontend` | `runtime: static` | 静态站点，SPA 路由重写，`/api/*` 反向代理至后端 |
| `njuatlas-backend` | `runtime: python` | Flask + Gunicorn（`gthread` worker），`rootDir: backend` |
| `njuatlas-redis` | Key Value | Flask-Limiter 存储与 SSE Pub/Sub |

后端启动命令（已配置）：

```bash
flask db upgrade && gunicorn -c gunicorn.conf.py "app:create_app()"
```

须在 Render Dashboard 手动配置的密钥类环境变量（`sync: false`）：

`SECRET_KEY`、`DATABASE_URL`、`GAODE_API_KEY`、`ZHIPU_API_KEY`、`BAILIAN_API_KEY`、`RESEND_API_KEY`、`MAIL_FROM`

`render.yaml` 已为生产后端写入的业务配置（随仓库部署，一般无需在 Dashboard 重复设置）：

| 变量 | 生产值 | 说明 |
|------|--------|------|
| `JWT_EXPIRATION_SECONDS` | `604800` | Access Token 有效期 7 天 |
| `REGISTER_EMAIL_RESTRICTION_ENABLED` | `false` | 允许任意邮箱注册 |

若 Dashboard 中曾手动设置过同名变量，会覆盖 Blueprint 值；以实际 `/api/user/auth-config` 与登录返回的 `expires_in` 为准。

Python 版本可通过环境变量 `PYTHON_VERSION`（如 `3.12`）或 `backend/.python-version` 指定，参见 Render Flask 部署文档（https://render.com/docs/deploy-flask）。

部署流程：

1. 关联 Git 仓库 → **New → Blueprint** → 选择含 `render.yaml` 的分支。
2. 为 `sync: false` 变量填入生产值；`DATABASE_URL` 使用 Neon 池化连接串（含 `sslmode=require`）。
3. 构建完成后通过 `/api/health` 健康检查验证。
4. 默认分配 `*.onrender.com` 子域，可绑定自定义域名。

### 3.2 阿里云 DNS 与 Render 对接

域名托管于阿里云时，典型解析配置如下：

| 记录类型 | 主机记录 | 记录值 | 说明 |
|----------|----------|--------|------|
| **CNAME** | `www` | `<frontend>.onrender.com` | 前端子域指向 Render 静态服务 |
| **CNAME** | `api` | `<backend>.onrender.com` | API 子域指向 Render Web Service |
| **ALIAS/ANAME 或 A** | `@` | `216.24.57.1` 或等效 CNAME 扁平化 | 根域指向 Render 负载均衡（参见 Render DNS 配置文档：https://render.com/docs/configure-other-dns） |

标准 DNS A/CNAME 记录仅解析至 IP 或主机名，不包含端口信息；HTTPS 终止由 Render 平台处理，无需自建反向代理。复杂路由需求可参考阿里云 ECS + Nginx 方案，本项目当前未采用。

在 Render Dashboard → **Custom Domains** 添加 `njuatlas.cn` 与 `api.njuatlas.cn`，按指引完成 DNS 验证与 TLS 证书签发。

### 3.3 生产域名

| 服务 | 域名 |
|------|------|
| 前端站点 | `https://njuatlas.cn` |
| 后端 API | `https://api.njuatlas.cn` |

## 五、常见问题排查

| 现象 | 可能原因 | 处理建议 |
|------|----------|----------|
| `无法加载 start.ps1` | PowerShell ExecutionPolicy 限制 | `Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass` |
| 端口 5000/8080 占用 | 端口冲突 | 终止占用进程，或修改 `run.py` / `http.server` 监听端口 |
| 数据库连接失败 / `connection requires SSL` | 连接串缺少 TLS 参数 | 追加 `?sslmode=require` |
| 迁移报错 `prepared statement already exists` | 经 PgBouncer 池化连接执行 DDL | 改用 Neon 直连 URL 执行 `flask db upgrade` |
| 本地页面可访问但鉴权失败 | 前端仍请求生产 API | 设置 `USE_LOCAL_API = true` 并确认本地后端已启动 |
| AI 接口无响应 | LLM API Key 未配置 | 在 `.env` 配置 `BAILIAN_API_KEY` 或 `ZHIPU_API_KEY` |
| 地图组件未渲染 | 高德 Key 缺失 | 在 `index.html` 的 `NJUATLAS_CONFIG.AMAP_KEY` 或 `js/amap.local.js` 中配置 |
