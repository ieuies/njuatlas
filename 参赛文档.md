# njuatlas.cn —— 让校园生活更便捷

## 一、作品介绍

**NJUAtlas（南大图谱）** 是面向南京大学学生的校园生活一体化 Web 平台，生产环境已部署于 [https://njuatlas.cn](https://njuatlas.cn)。系统整合周边生活服务信息检索、校园组局发布与匹配、AI 辅助推荐及轻量社交通信四类能力，覆盖鼓楼、仙林、浦口、苏州四个校区场景。

当前校园内组局需求分散于微信群、QQ 群及表白墙等渠道，信息检索成本高、状态不可追踪、参与路径不明确。项目组在立项前对校内相关渠道进行了调研：表白墙单日可见多条找搭子帖，部分 QQ 群亦持续出现同类消息，但回复率偏低，发帖者难以确认是否已匹配成功，潜在参与者也无法判断活动是否仍开放；信息随时间推移被新内容覆盖，供需双方均存在明显信息差。

NJUAtlas 将上述需求收敛至统一入口：「找搭子」模块支持分类浏览与活动发布；「吃喝玩乐」模块提供校区周边场所排行与详情；「AI 助手」支持自然语言查询（如「仙林附近川菜推荐」）；「个人中心」管理用户资料、收藏与对话历史；「消息」模块承载好友私信与互动通知。

技术架构为 **Flask 后端 + JavaScript SPA 前端**，数据层采用 **PostgreSQL（Neon 托管）**，应用部署于 **Render**，域名经 **阿里云 DNS** 解析至 Render 服务。

## 二、作品目标

1. **整合校园周边信息**：收录四校区餐饮、咖啡、娱乐、运动、商场等场所，结合高德地图 POI 与用户 UGC 点评，提供可筛选、可收藏的本地生活索引。
2. **降低组局匹配成本**：支持九类组局（饭搭子、运动、学习等）、地图可视化、报名参与与基于互动权重的热度排序。
3. **提供 AI 辅助决策**：接入智谱 GLM、阿里云百炼、DeepSeek 等大模型，结合用户行为偏好与高德 POI 数据，以对话形式输出餐饮与去处推荐。
4. **构建轻量校园社交层**：实现好友关系、私信、通知、个人主页及聊天气泡样式等能力，支撑组局匹配后的后续沟通。
5. **保障可部署性与可复现性**：提供 `start.ps1` 本地启动脚本、`render.yaml` 基础设施蓝图及完整环境变量模板，支持本地运行与云端一键部署。

### ⚠️ 部分完成 / 已知问题

| 问题 | 表现说明 |
|------|----------|
| **AI 模块数据提取不完善** | AI 路由含大量关键词/POI 类型匹配逻辑（`llm_routes.py` 中 `_collect_amap_pois`、严格 token 过滤等），复杂或口语化提问时，推荐店铺可能不全或不够精准；非餐饮类意图的识别仍有边界情况；无法准确回答商场中的店铺。 |
| **网页优化待改进** | 大模块（找搭子、指南、个人中心等）虽已懒加载，但首屏资源、移动端布局、部分页面动画仍在持续优化中；生产环境 API 与前端分域，存在跨域请求开销。 |
| **好友系统略有卡顿** | 团队已多次提交「消息页合并接口提速」「禁 memory 模式 SSE」「扩大连接池」等修复；在好友列表较多或频繁切换会话时，仍可能出现短暂加载延迟。 |
| **同域 API 反代未完全切换** | `render.yaml` 中 `/api/*` 反代规则已配置，但 `js/config.js` 生产环境仍指向 `https://api.njuatlas.cn/api`（历史兼容），同域代理方案待验证后统一。 |
| **个人中心列表未分页** | 收藏/点赞/评论列表接口暂无分页，数据量大时响应变慢（`WORKLOG_AND_NEXT_STEPS.md` 已记录）。 |
| **自动化测试缺失** | 仓库内无单元测试/集成测试目录，回归主要依赖人工验证。 |
